- 기본 파일명: Team1_report

  - Team1_report.hwp

- 허용 형식: hwp / docx / pdf / tex
- hwp는 별도 양식이 없으므로 docx 양식을 참조해 작성

## 3) 제출 기간

- 최종 보고서: 2026-06-12(금) 23:59까지

## 4) 분량 및 작성 규칙

- 본문 분량은 최대 20페이지 이하
- 참고문헌(References)은 페이지 수 산정에서 제외
- 템플릿 내용을 준용하되, 필요 시 subsection 또는 추가 섹션 기입 가능
- 그림/시각화가 많아 공간이 필요한 경우에 한해 extra +5페이지까지 허용

## 5) 권장 보고서 구조

최종 제출 보고서는 일반적인 과제 보고서 형식보다, 간결한 **저널 페이퍼 형식**을 준용하여 작성하는 것을 권장합니다. 기존 템플릿의 큰 흐름을 유지하되, 아래와 같은 구조를 참고하여 내용을 구체화하세요.

- Title
- Abstract
- Keywords
- 1. Introduction
  - 문제 배경과 필요성
  - 해결하고자 하는 과제 정의
  - 프로젝트 목표 및 기여 요약
- 2. Related Work
  - 기존 연구/유사 접근 방식 조사
  - 기존 방법과 본 프로젝트의 차이점 정리
- 3. Approach (or Method)
  - 전체 문제 해결 전략
  - 데이터 전처리 방법
  - 사용 모델 및 알고리즘 설명
  - 시스템/파이프라인 구성
- 4. Experiment
  - 데이터셋 설명
  - 실험 환경 및 설정
  - 학습/검증/테스트 분할 방식
  - 평가 지표 정의
- 5. Results
  - 주요 정량 결과 제시
  - 표/그림/시각화를 활용한 비교
- 6. Analysis / Discussion
  - 결과 해석
  - 성능이 잘 나온/안 나온 이유 분석
  - 실패 사례, 한계점, 개선 방향
- 7. Conclusion
  - 전체 프로젝트 요약
  - 기대 효과 및 향후 확장 가능성
- References

### 작성 시 유의사항

- 위 구조는 권장 양식이며, 프로젝트 특성에 따라 일부 섹션명을 조정하거나 통합할 수 있습니다.
- 예를 들어 **Related Work**가 적절하지 않은 주제라면, **Background** 또는 **Problem Setting**으로 대체 가능합니다.
- **Approach / Experiment / Results / Analysis**는 가능한 한 구분해서 작성하는 것을 권장합니다.
- 단순 진행 내용 나열이 아니라, **문제 정의 → 방법 → 실험 → 결과 → 해석**의 흐름이 드러나도록 작성하세요.
- 분량이 제한되어 있으므로 각 섹션은 핵심만 간결하게 요약하고, 표나 그림은 꼭 필요한 경우에만 포함하세요.
